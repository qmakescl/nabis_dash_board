# NABIS 대시보드 프로젝트 진행 상황 정리

본 문서는 지도 기반 균형발전지표 대시보드 구현을 위한 사전 준비 및 데이터 가공이 어디까지 진행되었는지 기록한 상태 저장 문서입니다. 다음 작업 착수 시 이 문서를 기준으로 이어나가시면 됩니다.

## 1. 개요 및 달성 목표
- **목표**: 국가균형발전종합정보시스템(NABIS) 통계 지표(핵심·객관/주관)를 대한민국 시·도 및 시·군·구 단위 지도에 시각화(Coropleth Map 등)하는 인터랙티브 대시보드 개발.
- **현재 진행 단계**: 전체 4단계(Phase) 중 **Phase 1 (데이터 분석 및 공간 데이터 전처리)** 까지 100% 완료되었습니다.

## 2. 완료된 주요 작업 (Phase 1)
1. **NABIS 웹 페이지 구조 확인 완료**
   - 핵심·객관지표 / 주관지표 메뉴 분리 구조 및 17개 시·도 - 시·군·구 드릴다운(jstree) UX 확인.
2. **개발 환경(Python 가상환경) 구축 완료**
   - `.venv` (Python 3.12 기반) 환경 및 `geopandas`, `pandas`, `topojson` 패키지 세팅 완료.
3. **V-World 기본 공간 데이터 분석 완료**
   - 시군구 경계 데이터(`BND_SIGUNGU_PG.shp`)와 '센서스 공간정보 지역 코드(Excel)'의 맵핑 적합성 검증 성공.
4. **💡 특이 행정구역 공간 병합(Dissolve) 처리에 성공**
   - *문제*: 수원시, 부천시, 창원시 등 12개 주요 '시'가 하위에 비자치구(행정 '구')를 가지고 있어 원본 지도 폴리곤이 나뉘어 있음. 화성시/남양주시는 구가 없음.
   - *해결*: 파이썬 스크립트(`process_shapefile.py`)를 통해 12개 시의 개별 구 폴리곤을 하나의 '시' 구역 단위로 말끔하게 병합 완료.
5. **📌 지도 데이터 웹 최적화(단순화) 완료** [매우 중요]
   - 원본 크기 **294MB**에 달하는 무거운 GeoJSON 파일을 TopoJSON 알고리즘으로 5km 수준의 스무딩 처리.
   - **3.6MB**로 용량이 1/100 가량 획기적으로 줄어든 웹 최적화 공간 데이터(`smooth_sgg_2025.json`) 추출 완료.

## 4. 향후 진행 예정 사항 (Next Steps: Phase 2~)
